WB Button Variants — Tag Syntax

The same buttons as the class-based demo, written entirely with <wb-button> custom-element tags — no class="wb-button ..." anywhere. Styling comes from wb-button[variant] / wb-button[size] attribute selectors.

💾 The Savings — why <wb-button> beats classes

The same button, both ways:

Class syntax · 81 chars
<button class="wb-button wb-button--primary wb-button--lg">Large Primary</button>
Tag syntax · 64 chars
<wb-button variant="primary" size="lg">Large Primary</wb-button>
21%
less markup per button (81 → 64 chars)
22%
smaller across a 9-button block (711 → 558 bytes)
0
repeated wb-button-- prefixes to type

No wb-button base class to remember, and no wb-button-- prefix on every modifier — variant and size are just attributes.
