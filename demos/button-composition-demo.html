WB Button — Composition with x-* behaviors

Same <wb-button> tags as before, now with x-* attributes layered on. Behaviors compose: each x- attribute adds one capability to the element — stack as many as you want, in any order, with no wrapper components and no JavaScript.

3 · Compose several behaviors on one button

Stack multiple x- attributes — they layer, they don't conflict:

Save (ripple + tooltip + toast) Delete (ripple + wobble + toast) Copy install cmd

The point — composition, not configuration

Base button
<wb-button variant="primary">Save</wb-button>
Same button, three behaviors composed
<wb-button variant="primary"
  x-ripple
  x-tooltip="Ripple + tooltip + toast" position="top"
  x-toast message="Saved!" type="success">
  Save
</wb-button>

No <RippleButton>, <TooltipButton>, or prop-drilling — just additive attributes. Any behavior works on any element, so the same x-ripple / x-tooltip you put on a button also works on a card, an image, or a <div>.
