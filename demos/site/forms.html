🧩 wb-button + x-* behavior composition

Any x-* behavior (ripple, tooltip, toast, etc.) composes onto a button the same way it does onto any other element — stack as many as you want, in any order, with no wrapper components.

Each x- attribute adds one capability to the button — no JavaScript.

Ripple Tooltip Toast on click Copy to clipboard Wobble Tada
  • x-ripplematerial-style ripple on click
  • x-tooltiphover hint, positionable
  • x-toastfires a toast notification
  • x-copycopies text to the clipboard
  • x-wobblewobble animation
  • x-tadatada celebration animation

Stack multiple x- attributes on one button — they layer, they don't conflict.

🧩 x-help — permanent hint text

x-help is for hint text a user should see without hovering or interacting — a short instruction that sits permanently under a field ("must be 8+ characters"). If the text should only appear on hover/focus, use x-tooltip instead — the two are deliberately kept separate.

Find this in your account settings under "Developer".
